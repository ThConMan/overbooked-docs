Anvil
Ascendant.

A bounded unsafe-enchantment system that makes endgame gear exciting without turning a co-op server into an economy wrecking ball.

How it works

Put two identical, single-enchantment books of the same level in an anvil. Once they are at the vanilla maximum, they combine into the next level. Each tier consumes two books and charges more XP, so higher tiers stay genuinely earned.

Sharpness V + Sharpness V → Sharpness VI

Default costs are 12, 20, 32, and 48 levels for successive unsafe tiers. Server owners can change every cap and cost in config.yml.

Two modes, one ruleset

DEFAULT

Simple mode

Set custom-station.enabled: false. No resource pack, special recipe, or menus. Every normal anvil can create and apply legal unsafe enchantments.

OPTIONAL

Enhanced mode

Set custom-station.enabled: true. Players craft a Netherite Infusion Anvil for a dedicated normal-anvil, unsafe-infusion, and name-styling interface.

Netherite Infusion Anvil

The station is intentionally expensive but reachable in co-op survival. It is indestructible in use: the netherite infusion prevents the normal anvil wear-and-break cycle.

Enhanced mode uses an optional server resource pack for the custom station model and icon. If you do not want to host or require a pack, leave enhanced mode off—nothing about the unsafe-enchant system is lost.

Safe power defaults

  • Sharpness, Smite, Bane, Impaling, and Power: up to +2 levels.
  • Protection variants: up to +1 level.
  • Efficiency, Unbreaking, and Feather Falling: up to +2 levels.
  • Fortune, Looting, Luck of the Sea, and Mending remain disabled by default.

Name styling in enhanced mode

Style a renamed item with colors, preset gradients, and Unicode Small Caps—the no-resource-pack style popularized by servers such as DonutSMP. The custom MiniMessage action accepts safe colors, hex colors, gradients, and text decorations only; it rejects interactive tags.

Install

  1. Run a current Paper server on Java 25.
  2. Copy the Anvil Ascendant JAR into plugins/.
  3. Start once, then edit plugins/AnvilAscendant/config.yml.
  4. Use /anvilascendant reload after configuration changes.
